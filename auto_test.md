# 嵌入式软件开发任务模板

## 一、软件架构概述
### 核心设计模式
1. **全局设备结构体**：`tMyDev TMyDev` 集中管理所有状态
2. **非阻塞时间驱动状态机**：`get_local_absolute_time()` 超时比较，禁止 delay()
3. **主状态机 + 子状态机**：`soft_state` 枚举分发到各业务模块的子状态机
4. **TLV通信协议**：AA帧头 + 设备类型 + 帧序列 + 消息类型 + TLV数据 + 校验
5. **Flash参数持久化**：阈值、SN、混液标志、基准峰峰值等关键参数掉电保存

### 硬件平台
1. STM32F103RBT6
2. IAR IDE

### 验收测试
1. 使用RTT进行验收，通过读取RTT串口输出，查看各功能是否正常
2. 编译0 Error 0 Warning
3. 下载成功

## 二、开发流程
1. 编译，下载，RTT验收脚本在scripts目录下，分别对应auto_test.bat, download.bat, verify_rtt.bat
2. 配置编译参数，确保编译环境正确设置
3. 编译工程，确保0 Error 0 Warning
4. 烧写工程，确保下载成功，无错误
5. 验收测试，通过RTT串口输出，确保各功能正常运行，检查.map文件查看内存占用情况
6. git提交代码，确保代码版本管理规范
7. 分解的任务，每个任务都按照以上流程进行开发和测试，确保每个功能模块都经过严格验证后再进行集成测试。

## 三、驱动接口
### 接口定义
1. PWM输出：PB6 1KHz  PB7 1KHz
2. ADC采集：PB0 ADC1_IN8 PB1 ADC1_IN9


## 四、任务分解
### 1. 工作状态
初始化 空闲 运行
### 2. 初始化状态
1. 配置PWM输出
2. 配置ADC采集
3. 设置默认参数
### 3. 空闲状态
1. 初始化完成后，定时10s进入空闲状态
### 4. 运行状态
1. 空闲状态下，定时10s进入运行状态
### 5. 运行状态
1. 运行状态下，定时10s输出PWM信号（PB6）, 50ms后，采集ADC信号
2. 采集到的ADC信号, 判断是不是0～3.3V
3. 如果是0～3.3V，则认为正常，如果不是，则认为异常
4. 关闭PWM信号（PB6）并打开PWM信号（PB7），50ms后，采集ADC信号
5. 采集到的ADC信号, 判断是不是0～3.3V
6. 如果是0～3.3V，则认为正常，如果不是，则认为异常
7. 关闭PWM信号（PB7）并打开PWM信号（PB6），50ms后，采集ADC信号
8. 采集完成后，定时10s进入空闲状态
